# java_laba5

## Простой контейнер внедрения зависимостей на основе рефлексии в Java

Учебный проект, демонстрирующий базовые принципы внедрения зависимостей с использованием Java Reflection API. Класс `Injector` автоматически находит поля, помеченные аннотацией `@AutoInjectable`, и подставляет нужные реализации, прочитанные из конфигурационного файла `.properties`. Проект помогает понять, как на низком уровне работают такие фреймворки, как Spring, и как рефлексия позволяет писать гибкий и расширяемый код.

## Возможности

-  Автоматическое внедрение зависимостей через аннотацию `@AutoInjectable`
-  Чтение маппинга «интерфейс → реализация» из внешнего `.properties`-файла
-  Использование Reflection API для доступа к приватным полям и установки значений
-  Простой интерфейс `ConfigurationProvider` для лёгкой замены источника настроек
-  Поддержка любых классов и интерфейсов без жёсткой привязки к конкретным типам
-  Минимальная конфигурация: достаточно указать соответствия в файле настроек
-  Отсутствие внешних зависимостей — только стандартная библиотека Java

## Требования

-  JDK 17+
-  Maven (для сборки и запуска проекта)
-  Конфигурационный файл `config.properties` с маппингом интерфейсов на реализации (пример есть в репозитории)
